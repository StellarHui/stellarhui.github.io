计算机网络（4）：网络层

发表于2025-11-17|更新于2026-01-13|计算机网络
|总字数:4.5k|阅读时长:13分钟|浏览量:

与链路层不同，网络层主要负责将数据包从源主机传送到目的主机，实现端到端传输。它关注的是跨越多跳的路由和转发（需要了解网络拓扑并计算最佳路径，采用存储-转发包交换技术：路由器接收完整数据包并校验后再转发）。

网络层服务模型

基础常识

异构互联网络

异构网络互连指的是将不同的网络（数据链路层和物理层至少有1个不同）使用中继系统进行连接，中继系统是：

  • 物理层中继系统：转发器，集线器。
  • 数据链路层中继系统：网桥或交换机。
  • 网络层中继系统：路由器。
  • 网络层以上的中继系统：网关。

考点：每个中继系统只能处理下层的异构。例如，多个局域网使用路由器相连，要求网络层以上的协议必须相同，因为路由器只包含：物理层、数据链路层、网络层。

因此，不难理解：同一网段内PC的网络号相同，不涉及网络层的IP地址转化，因此发送方会通过ARP 协议（IPv4）/NDP 协议（IPv6），获取对方的 MAC 地址，然后数据包会封装成 “以太网帧”，直接通过 交换机（二层设备） 转发，不会经过路由器。路由器是三层设备，核心作用是 “跨网段转发 IP 数据包”。

软件定义网络

分离数据平面与控制平面。

分组交换网根据其向端点系统提供的服务，可以分为2类：（1）面向连接的虚电路服务；（2）无连接的数据包服务。

面向连接的虚电路服务

与电路交换类似，PC之间进行通信时，先进行网络层的连接：建立逻辑上的虚链路（VC, virtual circuit），之后所有包沿同一路径传输。

为什么称为“虚链路”？因为它是通过沿途网络节点的转发表项建立的逻辑连接。同一条物理链路可以同时承载多条不同的虚电路，这些虚电路的数据包以复用的方式共享物理链路的传输能力。

这种服务分为3个步骤：

  • 虚链路建立。建立P2P的通信链路，然后将一个未使用过的虚电路号（VCID）分配给该链路。每个数据包只需包含数据内容+VCID。
  • 数据传输。双方就沿着该虚电路传输分组（数据包）。
  • 虚电路释放。PC之间通过释放请求来逐段拆除虚电路。

注：VCID管理机制。由于每个路由器只维护本地有意义的标识符，因此可能会触发VCID的冲突，为此，路由器在转发包时会替换连接标识符。

如图，H1和H3都使用了VCID=1，如果A都按照VCID=1转发，那么C将无法区分数据包来自H1还是H3。因此，A会将来自H1和H3数据包的VCID分别替换为1和2，以让C得以区分。

虚链路易错点：

  1. 虚链路也可以永久维持；
  2. 虚链路依赖于沿途各网络节点转发数据包实现，因此也采用分组交换、存储转发技术。
  3. 虚链路是可靠连接，因此不需要依赖其他协议就可以实现差错控制。
  4. 建立虚链路时，数据包需要带上源IP地址、目的IP地址，且需要进行路由选择。但是不需要与分配带宽（复用）。

无连接的数据包服务

如上图，这种服务中，每个数据包包含数据报文内容+IP地址（标识终点，下图中以A-F来表示）。路由器负责：（1）路由选择：为所有情况的分组，确定其下一跳的路径。动态维护更新路由表。（2）分组转发：当某一个特定分组到达时，采取什么操作。转发表查询、转发操作、队列管理和任务调度。

（注意，路由表可以动态更新）

两种服务的对比

路由算法与路由协议

下面的问题是：路由表怎么设计？这就需要使用路由选择协议。这些协议的核心是路由算法，即使用什么算法获取路由表中的各个条目。

路由算法

路由算法的目的是：如何为源路由器到目的路由器的通信建立“最佳”传输路径？（通常，“最佳”指通信开销最低，故通常是最短路算法）

路由算法的分类

  • 静态路由算法：固定的路由表。
  • 动态路由算法：根据网络流量负载和拓扑结构的变化动态调整路由表。

距离-向量路由算法

基于Bellman-Ford算法（方程）：可以这样设计路由算法：

  • 每个路由器维护自己的距离向量（即路由表），包含到所有已知目的地的距离和下一跳。
  • 每个路由器定期与邻居交换自己的整个距离向量。
  • 每个路由器收到邻居的距离向量后，根据上述公式重新计算自己的路由表。如果计算结果有变化，则更新自己的路由表，并在下一个周期通告给邻居。

一直执行上述过程，直到所有路由器的距离向量不再变化，即“收敛”。

距离-向量路由算法的问题：

  1. 慢收敛。当链路故障（坏消息）时，需要多次迭代才能让所有路由器知道。
  2. 计数到无穷。路由环路可能导致路由器之间一直更新路由表无法停下。

链路状态路由算法

还可以使用Dijkstra最短路径算法来确定路由表。具体步骤如下：

  • 每个路由器通过洪泛（Flooding），向全网所有路由器发送自己的链路状态通告（LSA, 即Link-State Advertisement，描述它与所有邻居的通信延迟）。

洪泛（Flooding）：实际上是一种广播。其实现方法是：每个路由器收到LSA之后，检查是否第一次收到该LSA（每个LSA被附上一个序列号作为独立标识）。若不是，则向所有邻居转发；否则不转发。

  • 所有路由器最终会收集到全网的LSA，形成一个完全一致的链路状态数据库（即）。
  • 每个路由器以自己为根，在这个统一的数据库上独立运行Dijkstra最短路径算法，计算出一棵最短路径树。并以这棵树构建自己的路由表。

分层路由算法

有时候整个网络规模非常大，导致路由表巨大。可以采用分层路由的算法：即路由表中的每一项只记录到达每个“自治系统”（AS）的下一跳和距离。

据此，可以将路由选择协议分为2部分：

  • 内部网关协议 (Interior Gateway Protocol)：AS内部的路由协议。
  • 外部网关协议 (External Gateway Protocol)：AS之间的路由协议。

下面，基于上面的几种路由算法，我们介绍几种路由协议。

路由协议

路由信息协议RIP

RIP属于内部网关协议（IGP），其基于距离-向量路由算法。

路由表：每个路由器保存自己到其余目标网络（或路由器）的距离，即距离向量。路由表每一项的格式如下：<目标网络名称，距离，下一跳路由器>。记路由表为向量

其中，，16表示不可达，即最大的距离为15。这是为了防止环路路由带来的“计数到无穷”问题。

协议内容：对路由器，

  • 发送RIP报文：每隔时间或者拓扑发生变化（自身路由表发生变化）时，向它的邻居发送它的路由表。
  • 收到RIP报文：当路由器收到路由表时，先将其修改为遍历其每一行，进行如下处理：

流量管理

常见的解决方案：

它们都可以从不同的角度解决问题。下面介绍一种路由器之间带宽分配的方法：Max-min Fairness（最大化最小流），即，在保证每个流都获得通信带宽的前提下，尽可能最大化获得最少带宽的流所享受的带宽。

这种分配方法达到的最终效果是：任何一个流都必须通过减少其他流的带宽来增加自己享受的带宽（带宽分配无法更优）。

注意，每个flow享受到的有效带宽是它所经过的所有路由器-路由器链路上最小的带宽。最大化的对象应当是所有有效带宽的最小值！因此，针对flow A，上图中R1, R2之间的带宽并没有分配到1，而是仅分配2/3。

在PPT中，这属于下一章《传输层》的内容。但个人认为这个流量控制应当放到网络层讨论。

网络地址转换（NAT）

允许私有IP地址被LAN重复使用。3个私有IP地址块如下，仅用于LAN（并且不同LAN之间可以复用），不可以用于WAN。具体的实现依靠NAT路由器，它可以通过NAT转换表来改变（传输层）数据包的IP地址和端口号。

互联网的路由器对目的地址是私有地址的IP 数据报一律不进行转发。有3个私有地址段：

  • 10.0.0.0/8，即10.0.0.0~10.255.255.255，相当于1个A类网络。
  • 172.16.0.0/12，即172.16.0.0~172.31.255.255，相当于16个连续的B类网络。
  • 192.168.0.0/16，即192.168.0.0~192.168.255.255，相当于256个连续的C类网络。

划分子网、子网掩码与默认网关

子网掩码指明IP地址的主机号哪些部分被用来划分子网，0对应主机号。发送数据包时，若目标地址不在本子网中，需要转发到默认网关（路由器）。

注：设有位主机号用来划分子网，那么这个子网至多有个主机，因为全0和全1的主机号分别用来标识子网和广播。

无分类编址CIDR

其实和子网掩码的思想非常类似。形如：128.14.32.5/20，表示子网掩码的前20位是1，后面均为0。

128.14.32.5/20表示一个CIDR地址块，其还可以再进行子网划分：在剩下的12位中选取前面位来作为子网号。

路由聚合

在路由表中使用较大的一个CIDR地址块来代替许多较小的地址块，节省空间。

网络层控制协议

网络层转发分组的过程

查找路由表项时遵循“最长前缀匹配原则”。路由器执行的分组转发算法如下：

  1. 从收到的IP分组的首部提取目的主机的IP地址（目的地址）。
  2. 若查找到特定主机路由（目的地址为），则按照这条路由的下一跳转发分组；否则从转发表中的下一条（按前缀长度的顺序，从长到短查询）开始检查，执行步骤3）。
  3. 将这一行的子网掩码与目的地址逐位执行（按位与）操作。若运算结果与本行的前缀匹配，则查找结束，按照“下一跳”指出的进行处理（或者直接交付本网络上的目的主机，或通过指定接口发送到下一跳路由器）。否则，若转发表还有下一行，则对下一行进行检查，重新执行步骤3）。否则，执行步骤4）。
  4. 若转发表中有一个默认路由，则把分组传送给默认路由；否则，报告转发分组出错。

动态主机配置协议DHCP

DHCP: Dynamic Host Configuration Protocol.

DHCP服务器和DHCP客户的交换过程如下:

  1. DHCP 客户广播“DHCP 发现”消息，试图找到网络中的 DHCP服务器，以便从 DHCP 服务器获得一个IP地址。源地址为0.0.0.0，目的地址为255.255.255.255。
  2. DHCP 服务器收到“DHCP 发现”消息后，广播“DHCP 提供”消息，其中包括提供给 DHCP 客户机的IP地址。源地址为 DHCP 服务器地址，目的地址为255.255.255.255。
  3. DHCP 客户收到“DHCP 提供”消息，若接受该 IP地址，则广播“DHCP 请求”消息，向 DHCP服务器请求提供IP地址。源地址为0.0.0.0，目的地址为255.255.255.255。
  4. DHCP服务器广播“DHCP 确认”消息，将IP地址分配给 DHCP 客户。源地址为 DHCP 服务器地址，目的地址为255.255.255.255。

注意：这是一个应用层协议！

网际控制报文协议ICMP

ICMP: Internet Control Message Protocol.

ICMP 报文有两种，即ICMP 差错报告报文和ICMP 询问报文。

ping在应用层，使用网络层ICMP协议的回送请求和回答报文；traceroute在网络层，使用网络层ICMP协议的时间超过报文。
